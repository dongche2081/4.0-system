# 4.0-system 项目规范

## 技术栈约束
- 框架：React 18 + TypeScript（严格模式）
- 样式：Tailwind CSS，主色调 #F2C94C（黄色），禁止紫色/蓝色渐变
- 路由：React Router v6，页面组件必须用 React.lazy() 包裹
- 状态：全局状态必须用 useApp()（来自 contexts/AppContext），禁止在组件内用 useState 存全局数据

## 架构规则
- 页面组件：src/pages/*.tsx，负责路由和数据获取
- 业务组件：src/components/domain/*/，按领域分组（diagnose/ expert/ practice/）
- 工具组件：src/components/ui/，纯展示无逻辑
- 包装器：src/components/wrappers/，专门处理路由参数转换

## 命名规范
- xxxEngine.tsx：复杂业务逻辑（DiagnoseEngine, SimulationEngine）
- xxxCard.tsx：展示卡片（ExpertCard, DigestCard）
- xxxView.tsx：页面级视图（HistoryView, ExpertProfileView）
- 禁止：Page.tsx 后缀（用 View 或直接路由名）

## 业务术语（强制使用）
- Diagnose：诊断流程（多步骤问卷）
- Prescription：AI 生成的管理建议（包含 truth/script/redLines）
- Simulation：实战演练（选择题场景）
- TacticalBriefing：战术简报（话题详情展示）
- Context：用户画像（业务阶段、团队状态等）

## 代码生成约束
1. 所有 API 调用必须通过 services/ 层，禁止在组件内直接 fetch
2. 所有 localStorage 操作必须通过 useAppState，禁止直接 localStorage.setItem
3. 动画使用 motion/react，禁止生硬切换
4. 类型定义必须来自 src/types.ts，禁止内联定义 interface

## 调试要求
- 生成代码后自动在关键函数添加 console.log('[DEBUG] 函数名: 参数')
- 遇到路由跳转必须确认 navigate 路径与 Route path 匹配

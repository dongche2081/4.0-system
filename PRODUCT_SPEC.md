# AI 管理能力提升助手 —— 产品功能说明书（面向新成员）

> 本文档从功能视角介绍产品，帮助你快速理解「我们在做什么、为谁做、核心流程是什么」。不包含技术实现细节。

---

## 一、产品定位与一句话介绍

**AI 管理能力提升助手** 是一款面向中基层管理者的 AI 教练产品。它的核心目标是：

> **当管理者遇到具体的管理困境时，能在最短时间内获得「看得懂、用得上、可落地」的实战建议。**

我们不做长篇大论的管理课，而是像一位高冷的实战派顾问，针对你当下的问题，直接给本质判断、给谈话剧本、给动作红线。

### 目标用户画像
- 技术骨干刚转管理的新晋主管
- 带 5~30 人团队的中层管理者
- 正在经历快速扩张期或组织变革期的团队负责人

### 产品调性
- **语言风格**：军事化/指挥官风格（"战术选择"、"研判报告"、"战地指标"）
- **AI 人设**：高冷、专业、直指人心，不说教、不讨好、字字珠玑
- **品牌色**：琥珀金（#F2C94C）

## 二、核心使用流程（用户动线）

用户进入产品后，主要通过 **"学 → 诊 → 练 → 问"** 四个环节解决问题：

```
┌─────────────────────────────────────────────────────────────┐
│  用户遇到管理问题（如：核心骨干想离职）                        │
└──────────────────┬──────────────────────────────────────────┘
                   ▼
┌─────────────────────────────────────────────────────────────┐
│  1. 问一问（首页）                                            │
│     • 搜索问题，或从 TOP 10 榜单中找到相似痛点                │
│     • 查看话题详情：AI 解析本质原因 + 实战案例 + 匹配专家     │
└──────────────────┬──────────────────────────────────────────┘
                   ▼
┌─────────────────────────────────────────────────────────────┐
│  2. 聊一聊（深度诊断）                                        │
│     • 描述具体场景 → 回答 3 道针对性调研题                   │
│     • AI 生成个性化诊断报告（风险评分 + 谈话剧本 + 红线）     │
│     • 可继续追问，获得更细化的执行建议                        │
└──────────────────┬──────────────────────────────────────────┘
                   ▼
┌─────────────────────────────────────────────────────────────┐
│  3. 练一练（实战推演）                                        │
│     • 进入对应管理场景的选择题                                │
│     • 选择应对策略 → 即时反馈对错与深度解析                   │
│     • 提升决策肌肉记忆                                        │
└──────────────────┬──────────────────────────────────────────┘
                   ▼
┌─────────────────────────────────────────────────────────────┐
│  4. 历史记录                                                  │
│     • 随时回看学过的案例、练过的场景、聊过的诊断              │
│     • 支持从记录处继续追问或重练                              │
└─────────────────────────────────────────────────────────────┘
```

## 三、四大核心功能模块

### 3.1 问一问（首页 / 知识库）

这是用户的主入口。页面布局分为上下两部分：

#### 顶部：全局搜索
- 一个显眼的中央搜索框，提示语："今天有什么可以帮你？输入你的管理痛点，AI 将为你自动匹配自身管理者的实战经验"
- 搜索后进入话题详情页，或直接触发 AI 诊断

#### 底部：三栏内容区
| 栏目 | 内容说明 |
|------|---------|
| **个人高频痛点 TOP 10** | 管理者最常见、最痛的实战问题，如"下属执行力差，推一下动一下"、"核心骨干突然离职"等 |
| **公司热点话题 TOP 10** | 偏组织必修类话题，如"如何给下属做绩效面谈"、"从技术骨干向管理者的转型"等 |
| **专家榜** | 展示平台签约/入驻的管理专家，显示积分、内容数、标签等 |

#### 话题详情页里有什么？
每个话题点进去后，包含 5 张卡片：
1. **标题卡**：话题类型（战友最痛 / 组织必修）、浏览量、HOT 标识
2. **AI 智能解析**：基于真实企业场景撰写的实战案例，可展开/收起，支持复制、点赞、点踩、收藏、"我也来支招"
3. **参考专家**：AI 根据该话题和用户团队画像，自动匹配 3 位最合适的专家，可一键查看专家的音视频/文字案例
4. **实战转化**：两个入口——
   - **聊一聊**：针对该话题进入深度诊断流程
   - **练一练**：进入对应该话题的管理情景模拟题
5. **相关话题**：手动配置的关联话题，引导用户持续学习

---

### 3.2 聊一聊（AI 深度诊断）

这是产品最具差异化的功能。它不是简单的问答，而是**结构化的管理会诊**。

#### 三步流程
1. **描述场景**：用户输入自己的管理困境（也可从常见困境卡片快速选择）
2. **回答题目**：系统根据用户描述的关键词，智能匹配一套调研问卷
   - 如果提到"离职/人才" → 进入「人才保留」维度
   - 如果提到"执行力/推不动" → 进入「执行力穿透」维度
   - 其他 → 进入「基础组织画像」维度
   - 每套问卷约 3 道题，包含单选和补充细节文本框
3. **AI 生成**：生成一份结构化的《研判报告》

#### 诊断报告包含什么？
| 模块 | 说明 |
|------|------|
| **风险评分** | 0-100 分的量化评估，分为低危/中危/高危/极高危 |
| **研判真相** | 一针见血指出问题本质（如：执行力差的本质是目标拆解与反馈机制断层） |
| **破局思路** | 核心建议的一句话总结 |
| **谈话剧本** | 可直接套用的对话模板：开场白 → 核心拆解要点 → 收尾动作 |
| **动作红线** | 绝对禁止做的几件事（如：严禁第一时间用加薪防御性挽留） |

#### 追问模式
用户在看完报告后，可以在底部输入框继续追问（如"具体的一对一沟通该怎么开场？"），AI 会结合之前的诊断上下文给出针对性补充。

#### 紧急预警
如果用户在个人画像中填写的"团队压力指数"超过 8 分，首页会出现红色浮动预警条：
> "紧急军事通报：监测到高压爆仓风险，建议立即阅研《扩张期人才防御建议》"

---

### 3.3 练一练（实战推演 / 情景模拟）

这是一个管理场景的"选择题闯关"系统，帮助用户把知识转化为决策直觉。

#### 场景库
目前覆盖了 8+ 个高频管理场景，每个场景都有标签分类：
- 绩效宣贯、离职预警、执行力差
- 跨部门沟通、团队冲突、目标对齐
- 新生代管理、向上管理 等

#### 单题交互流程
1. **阅读情境**：左半屏展示一个具体的管理困境描述（部分题目还配了组织架构图）
2. **战术选择**：右半屏呈现 2~3 个应对选项
3. **提交判定**：选择后即时显示结果
   - 选对 → "决策判定：推演成功" + 绿色标识
   - 选错 → "决策判定：红区警告" + 红色标识
4. **深度解析**：给出为什么对/为什么错的详细战术分析
5. **后续动作**：可收藏题目、退出、或进入下一关

#### 数据展示
每个场景卡片上会显示：
- **练习人数**：已有多少人做过这道题
- **正确率**：全平台的平均正确率

用户也可以按"练习人数"或"正确率"排序，或通过标签筛选特定类型场景。

---

### 3.4 历史记录

用户的学习行为被记录在三类档案中，方便复习和持续跟进：

| 分类 | 记录内容 | 关键信息 |
|------|---------|---------|
| **学一学** | 浏览过的话题详情页 | 话题标题、专家姓名、阅读时长、浏览/收藏行为 |
| **练一练** | 做过的情景模拟题 | 场景标题、答题用时、决策正误、对士气/效率/保留率的影响值 |
| **聊一聊** | 深度诊断记录 | 用户原始问题、AI 报告摘要、对话轮数、时间 |

#### 能力
- 搜索历史记录
- 按时间筛选（最近 7 天 / 30 天 / 全部）
- 单条删除或整类清空
- 点击记录可跳回对应页面继续学习/追问

## 四、专家系统

产品是 **"AI + 真人专家"** 的双轮驱动模式。

### 专家内容形态
每位专家有自己的人设主页，内容形态包括三种：
- **视频**：SOP 系列、实战复盘
- **音频**：原声节选、管理心得
- **文字**：核心复盘文档、深度案例分析

### 专家匹配逻辑
在话题详情页，系统会根据以下维度为用户推荐最合适的专家：
1. **话题领域匹配**（40%）：专家标签是否覆盖该话题
2. **管理风格匹配**（30%）：用户画像中的"管理模式"是否与专家背景契合（如华为实战模式 vs 互联网敏捷模式）
3. **业务阶段匹配**（30%）：用户所在"业务阶段"是否与专家实战经验一致（如快速扩张期、组织变革期）

### 专家预约
- 用户在专家主页可点击"预约咨询"
- 需要消耗 **300 战术积分**
- 积分不足时提示："战术积分不足，请通过实战推演获取积分。"
- 预约成功后提示："专家将在 24 小时内联系您。"

## 五、用户画像与积分体系

### 5.1 团队画像（ProfileContext）
用户登录后，可在个人中心完善自己的团队画像。这些数据会作为 AI 生成建议的上下文，让建议更精准：

| 维度 | 选项示例 |
|------|---------|
| 业务阶段 | 初创生存期 / 快速扩张期 / 稳定成熟期 / 组织变革期 |
| 团队状态 | 高压冲刺中 / 人心浮动中 / 内耗血栓化 / 人才紧缺中 |
| 管理风格 | 强势结果导向 / 温和关系导向 / 授权放权型 / 严密监控型 |
| 管理模式 | 华为实战模式 / 互联网敏捷模式 |
| 管理幅度 | 带多少人 |
| 环境压强 | 1-10 分的压力自评 |

### 5.2 战术积分
- 积分是产品内的"货币"，目前主要通过"实战推演"获取
- 用途：预约专家咨询
- 在个人中心首页醒目显示当前积分总额

### 5.3 我的贡献
用户可以对话题内容"支招"（提交自己的实战经验）。贡献被采纳后可获得积分。贡献列表会显示：
- 建议标题与内容
- 获得赞同数
- 获得积分
- 审核状态（已通过 / 审核中）

## 六、登录与数据机制

### 登录
- 目前采用简洁的前端登录形式：输入"管理 ID" + "通行密钥"即可进入
- 未登录用户只能看首页，点击其他模块会提示："指挥官，请先对一对身份（登录）以解锁该模块。"

### 数据存储
- 用户的历史记录、诊断报告、演练记录等优先存储在本地（localStorage）
- 系统会检测云端备份，并在每天首次打开时自动恢复最新数据
- 页面关闭前也会触发自动备份

## 七、内容运营体系

产品的内容分为两大类，共 10 个核心话题：

### 战友最痛（个人视角）
1. 下属执行力差，推一下动一下
2. 跨部门协作难，对方不配合
3. 核心骨干突然离职，如何应对
4. 新晋管理者如何建立威信
5. 会议效率低，议而不决

### 组织必修（组织视角）
6. 如何给下属做绩效面谈
7. 团队目标对齐与压力传递
8. 管理者的时间管理与授权
9. 如何识别与培养高潜人才
10. 从技术骨干向管理者的转型

每个话题都配备了：
- 一个真实企业场景的 **实战案例**
- 一份 **管理处方**（本质原因 + 谈话剧本 + 红线）
- 关联的 **实战推演题目**
- 关联的 **推荐专家**

## 八、新成员快速 FAQ

**Q1：这个产品和普通 AI 问答有什么区别？**
> 普通 AI 问答是泛泛而谈，我们是基于预设的管理知识库 + 结构化诊断问卷 + 用户团队画像，输出可直接落地的管理处方（包括谈话剧本和红线）。

**Q2："问一问"和"聊一聊"有什么区别？**
> "问一问"是先看现成的标准答案（案例 + 处方）；"聊一聊"是针对你的具体团队情况，做深度问诊，生成定制化报告。

**Q3：专家是真人吗？**
> 是的，专家是平台签约的实战派管理者（如前华为 OD 专家、字节 HRBP、阿里政委等）。用户可以查看他们的案例，也可以用积分预约一对一咨询。

**Q4：积分怎么获得？**
> 目前主要通过"练一练"（实战推演）获取，也可以通过"我也来支招"被采纳后获得奖励积分。

**Q5：如果 AI 服务不可用，产品还能用吗？**
> 能。所有话题案例、处方、推演题目都是预置内容，不依赖 AI 也能正常使用。只有"聊一聊"的深度追问和个性化报告生成需要调用 AI 服务，此时会降级为本地预设的兜底内容。
